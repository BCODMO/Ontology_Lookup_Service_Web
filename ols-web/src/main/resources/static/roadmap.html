Ontology Lookup Service version 2 retired

If you are looking for the older version of OLS this has now been officially retired and the service is no longer available. This page conatins some details about the existing OLS service and includes a guide for helping you migrate. Please contact us on ols-support [at] ebi.ac.uk if you have any questions.

Ontology Lookup Service version 3 roadmap

OLS 3 is a complete re-write of the code and includes some significant changes to how we index and serve up ontology data. This includes an all new API for programmatic access to the data. The old OLS 2 service is no longer able to update the ontologies and we plan to deprecate the service entirely in October 2016. This document outlines the road map for releasing the new OLS service and provides a guide for migrating from the old SOAP API to the new REST API.

Why is this happening?

The code under the old OLS system is nearly 10 years old in places and many of the libraries have been deprecated which made maintaining the codebase very difficult. The old OLS systems was only designed to work with OBO ontologies and used public version control systems, like the OBO foundry sourceforge CVS site, for updating the ontologies. This system started failing recently, partly becuase it could no longer cope with the increasing size of the ontologies and there were other factors like the deprecation of the OBO CVS site on sourceforge. We decide that it was time for a new OLS service that would be more scalable and easier to maintain. The main features of this service are:

  • Supports both OWL and OBO formatted ontologies
  • Ontology crawler that is based on registering files by URLs and checking those files nightly for changes
  • Built for scalability using two industry standard technologies - Neo4j and Solr

Timeline

  • October 27th 2015
    • OLS beta announced running at http://www.ebi.ac.uk/ols/beta
    • Announcement of deprecation of old OLS services running at http://www.ebi.ac.uk/ontology-lookup (also redirects from http://www.ebi.ac.uk/ols)
    • Freeze data in old OLS and stop nightly update of ontologies
  • January 31st 2016
    • We will start an instance of the old OLS at http://www.ebi.ac.uk/ontology-lookup/v2. This will serve the old services and data until October 1st 2016
  • May 4th 2016
    • We will move new OLS to production and serve from http://www.ebi.ac.uk/ols
    • http://www.ebi.ac.uk/ontology-lookup will redirect to http://www.ebi.ac.uk/ols
    • http://www.ebi.ac.uk/ols/beta will continue to be available for beta testing new features, but the underlying data may not be the most up-to-date
    • The old OLS service will be archived and only available from http://www.ebi.ac.uk/ols/v2
  • October 1st 2016
    • We will shutdown the old OLS service running at http://www.ebi.ac.uk/ols/v2

What will be deprecated

  • As of April 18 2016 we will no longer provide a MySQL dump of OLS
  • As of May 4th 2016 the SOAP API will no longer be accessible from http://www.ebi.ac.uk/ontology-lookup/OntologyQuery and only available from http://www.ebi.ac.uk/ols/v2/OntologyQuery until October 1st 2016

Migration

Linking to OLS

Linking to an ontology

If you want to link to an ontology in OLS you will need to know the ontology id. Typically this is a lowercase form of the ontology namespace e.g. For the gene ontology this is 'go'

Old URL New URL Example
www.ebi.ac.uk/ontology-lookup/browse.do?ontName=<ontology name> www.ebi.ac.uk/ols/ontologies/<ontology name> ebi.ac.uk/ols/ontologies/<ontology name> (Note that the live URL is currently for the beta site)

Linking to an ontology term

You can link to an ontology term in the old OLS using the OBO style short id e.g. GO:0007610. The new OLS moves to identify ontology terms via a International Resource Identifier (IRI). We still support linking via the old id style.

Old URL New URL Example
www.ebi.ac.uk/ontology-lookup/browse.do?ontName=<ontology name>&termId=GO%3A0007610 www.ebi.ac.uk/ols/ontologies/<ontology name>/terms?iri=<term =IRI> www.ebi.ac.uk/ols/ontologies/go/terms?iri=http%3A%2F%2Fpurl.obolibrary.org%2Fobo%2FGO_0007610 (Note that the live URL is currently for the beta site)
www.ebi.ac.uk/ols/ontologies/<ontology name>/terms?short_form=<term shortform> www.ebi.ac.uk/ols/ontologies/go/terms?short_form=GO_0007610 (Note that the live URL is currently for the beta site)
www.ebi.ac.uk/ols/ontologies/<ontology name>/terms?obo_id=<term obo id> www.ebi.ac.uk/ols/ontologies/go/terms?obo_id=GO:0007610 (Note that the live URL is currently for the beta site)

Using the OLS API

The old OLS API was built around SOAP, the WSDL for this service is described http://www.ebi.ac.uk/ontology-lookup/OntologyQuery.wsdl. The new OLS API is built around REST principles where individual resources are identified by URLs. You can find detailed documentation of the new OLS REST API here

Migrating from SOAP API methods to the new REST API

SOAP method Equivalent REST query
getAllTermsFromOntologyRequest api/ontologies/{ontology name}/terms
getOntologyLoadDateRequest api/ontologies/{ontology name}
getOntologyNamesRequest api/ontologies
getPrefixedTermsByNameRequest api/search?q=
getRootTermsRequest api/ontologies/{ontology name}/terms/roots
getTermByIdRequest api/ontologies/{ontology name}/terms?obo_id=
getTermChildrenRequest api/ontologies/{ontology name}/terms/{term IRI}/children
getTermMetadataRequest api/ontologies/{ontology name}/terms?obo_id=
getTermParentsRequest api/ontologies/{ontology name}/terms/{term IRI}/parents
getTermRelationsRequest api/ontologies/{ontology name}/terms?obo_id=
getTermsByAnnotationDataRequest api/search?q={query}&fl='annotation_{field name}'
getTermsByNameRequest api/search?q={query}&fl=label
getTermXrefsRequest api/search?q={query}&fl='annotation_{xref field name}'
isObsoleteRequest api/ontologies/{ontology name}/terms?obo_id=

Migrating from OLS Ajax API methods to the new Search API

Ajax call Equivalent REST query
ajax.view?q=termmetadata&termid={id}&ontologyname={ontology name} api/ontologies/{ontology name}/terms?obo_id={id}
ajax.view?q=termautocomplete&termname={label}&ontologyname={short name} api/select?q={label}&ontology_name={short name}
ajax.view?q=termname&termid={id}&ontologyname={ontology name} api/ontologies/{ontology name}/terms?obo_id={id}
